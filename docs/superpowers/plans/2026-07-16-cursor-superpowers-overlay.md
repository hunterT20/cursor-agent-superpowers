# Cursor Superpowers Overlay Implementation Plan

> **For agentic workers:** REQUIRED EXECUTION CONTRACT: the Codex/Superpowers controller owns this plan and every review. All implementation and review-fix edits must be performed by fresh `cursor-agent --model 'composer-2.5[fast=false]'` invocations. Cursor must not commit or mutate git history/remotes.

**Goal:** Build and verify four composable Codex skills that keep Superpowers in control while Cursor Agent implements project tasks.

**Architecture:** Four sibling skills form a narrow overlay over upstream Superpowers. `cursor-agent-bridge` provides the only executable boundary; `using-cursor-superpowers` routes requests; `executing-plans-with-cursor` manages sequential task dispatch; `reviewing-cursor-changes` owns controller review and fix loops. A Python standard-library test suite verifies the bridge and static cross-skill contracts.

**Tech Stack:** Markdown Agent Skills, Python 3 standard library, `unittest`, Git, Cursor Agent CLI, official Codex `skill-creator` scripts.

## Global Constraints

- Use the exact Cursor model `composer-2.5[fast=false]` for implementation and fix invocations.
- Superpowers/Codex owns discovery, design, planning, review, verification, and branch completion.
- Cursor may edit files and run tests only inside the selected task workspace and scope.
- Cursor must not commit, amend, push, merge, rebase, reset, tag, or switch branches.
- Codex must not edit implementation code while the overlay execution workflow is active.
- Execute one implementation task at a time in a worktree; never run parallel Cursor implementers in the same worktree.
- Treat Cursor reports as worker-supplied evidence, not proof; verify `HEAD`, diff, requirements, and tests independently.
- Do not copy the 14 upstream Superpowers skills.
- Use only `SKILL.md`, generated `agents/openai.yaml`, and directly required scripts/tests; do not add README or placeholder files.
